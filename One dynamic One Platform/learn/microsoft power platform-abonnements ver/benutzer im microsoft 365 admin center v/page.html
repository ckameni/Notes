Mehrere Benutzer hinzufügen
Sie können schnell und einfach mehrere Benutzer ohne das Formular mit den vier Registerkarten hinzufügen, indem Sie Users>Active users>Add multiple users auswählen.
 

Vorhandene Benutzer verwalten
Die Verwaltung der Rollen und Lizenzen Ihrer aktuellen Benutzer erfolgt einfach über das Verzeichnis Active users. Es enthält ein Suchfeld und Filtersteuerelemente, mit denen sich die Benutzer schnell sortieren lassen.

Weitere Informationen finden Sie unter Benutzer hinzufügen und Lizenzen zuweisen in Microsoft 365.

Wichtiger Hinweis zur Benutzererstellung und Lizenzzuweisung
Wenn Sie im Microsoft 365 Admin Center einen neuen Benutzer erstellen und ihm eine Lizenz zuweisen, beachten Sie die folgenden Punkte:

Synchronisation mit Kundenbindungsanwendungen
  • Der Benutzer wird automatisch auch in Kundenbindungsanwendungen (z. B. Dynamics 365) erstellt.
  • Synchronisationszeit: Der Abgleich zwischen dem Microsoft 365 Admin Center und den Kundenbindungsanwendungen kann einige Minuten in Anspruch nehmen.


Zugriffsrechte des Benutzers
  • Mit der Benutzer-ID und dem Kennwort kann der Benutzer:
    • Zugriff auf das Microsoft 365 Admin Center erhalten, um allgemeine Informationen zu Diensten anzuzeigen.
  • Der Benutzer hat jedoch keinen Zugriff auf Kundenbindungsanwendungen, bis:
    • Ihm direkt oder über eine Gruppe eine Sicherheitsrolle zugewiesen wurde.

Zusammengefasst:
  • Erstellen und Lizenzieren eines Benutzers allein reicht nicht aus, um Zugriff auf Anwendungen wie Dynamics 365 zu gewähren.
  • Stellen Sie sicher, dass der Benutzer eine passende Sicherheitsrolle erhält, um die gewünschten Funktionen nutzen zu können.
 
Überblick: Microsoft 365-Administratorrollen
Microsoft 365 bietet eine Vielzahl von Administratorrollen, die auf bestimmte Geschäfts- und Verwaltungsfunktionen abgestimmt sind. Jede Rolle gewährt spezifische Berechtigungen
im Microsoft 365 Admin Center. Im Folgenden werden die wichtigsten Rollen beschrieben, die sich besonders auf Microsoft Dynamics 365 und Power Platform beziehen.

1. Globaler Administrator
  • Funktion: Hat Zugriff auf alle administrativen Funktionen in der Microsoft 365-Suite (einschließlich Skype for Business).
  • Besonderheiten:
    • Kann andere Administratorrollen zuweisen und verwalten.
    • Diese Rolle hat die höchsten Berechtigungen und sollte sparsam vergeben werden (2 bis 4 Personen pro Organisation).
    • Für Power Platform-Entwickler ist es oft besser, die Rolle Power Platform-Administrator zu verwenden.

2. Power Platform-Administrator
  • Funktion:
    • Kann mehrere Umgebungen in der Power Platform verwalten.
    • Umfasst Systemadministratorrechte für Power Platform.
  • Empfohlen für: Power Platform-spezifische Verwaltungsaufgaben ohne Zugriff auf globale Einstellungen.

3. Abrechnungsadministrator
  • Funktion:
    • Verwalten von Käufen und Abonnements.
    • Erstellen von Supporttickets und Überwachen des Dienststatus.

4. Benutzeradministrator
  • Funktion:
    • Verwaltung von Benutzerkonten (z. B. Zurücksetzen von Kennwörtern, Hinzufügen oder Löschen von Benutzern).
    • Verwaltung von Supporttickets und Microsoft 365-Gruppen.
  • Einschränkungen:
    • Kann keine globalen Administratoren löschen oder verwalten.
    • Kann keine Kennwörter für Administratoren mit höheren Berechtigungen (z. B. Abrechnungs-, Exchange-, oder Compliance-Administratoren) zurücksetzen.

5. Dynamics 365-Administrator
  • Funktion:
    • Verwaltung von Dynamics 365-Instanzen und Mandantenebenen-Funktionen.
    • Administratorrechte in der Power Platform.
  • Einschränkungen:
    • Kein Zugriff auf globale Microsoft 365-Administratoraufgaben (z. B. Benutzerkonten- oder Abonnementverwaltung).
    • Kein Zugriff auf Einstellungen für Microsoft 365-Apps wie Exchange oder SharePoint.
Zusammenfassung der Rollenauswahl
  • Globaler Administrator: Für umfassende Kontrolle über die gesamte Microsoft 365-Umgebung (nur wenigen Personen zuweisen).
  • Power Platform-Administrator: Speziell für die Verwaltung der Power Platform.
  • Dynamics 365-Administrator: Ideal für die Verwaltung von Dynamics 365, ohne globale Berechtigungen zu vergeben.
  • Abrechnungs- und Benutzeradministratoren: Für spezifische Aufgaben wie Abonnementverwaltung oder Benutzerkontenpflege.

Mit dieser klaren Rollenverteilung können Sie sicherstellen, dass jede Person die benötigten Berechtigungen hat, ohne unnötige Risiken einzugehen.

Weitere Informationen finden Sie unter Berechtigungen der Administratorrolle in Microsoft Entra ID.
